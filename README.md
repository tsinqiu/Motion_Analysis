## 一、项目简介与目标

本项目是数据库课程大作业，主题为：

> 基于 Garmin 可穿戴设备数据的运动健康分析数据库系统设计与实现

系统以 Garmin 手表采集的运动数据为基础，围绕跑步、骑行等运动记录，完成数据导入、数据库设计、后端接口开发与前端可视化展示，实现对各指标的管理与分析。

采用前后端分离结构：

```text
Garmin 手表数据
    ↓
数据导入与清洗
    ↓
数据库存储
    ↓
后端 API 服务
    ↓
前端页面展示
    ↓
运动数据分析与可视化
```

> 前端不直接连接数据库，前端只调用后端 API，前期后端还未构建好API与连接时可以采用MOCK模拟数据测试；
> 后端负责数据库访问与业务逻辑；
> 数据库部分通过 SQL 脚本、数据字典和备份文件进行维护。

完成以下目标：

1. **数据库**：设计一个面向运动数据管理与分析的关系型数据库（MySQL）并将 Garmin 手表采集的运动数据导入；设计SQL语句，实现运动记录、运动详情、心率数据、运动统计等信息的存储与查询；
2. **后端** ：对数据库数据进行封装，可加入简单的机器学习进行数据分析，给前端提供API接口；
3. **前端**：通过前端页面(Vue)展示运动数据与各项可视化，效果参见cz的视频；
4. **上云**：完成后将后端与数据库上传到服务器（香橙派，若因IP问题无法解决可转为香港服务器）
5. **最终实现**：体现数据库课程中的表结构设计、主外键约束、多表关联查询、聚合统计、视图、索引等内容。
